## 1. Общее описание проекта

Проект представляет собой прототип трёхпалой роботизированной кисти с системой силомоментной обратной связи. Все три пальца активные и подвижные, каждый имеет две фаланги и независимый сервопривод. Кисть способна захватывать предметы различной формы — от цилиндрических до плоских — благодаря системе адаптивного схвата с контролем усилия.

**Ключевые особенности:**

- Три независимых пальца с двумя фалангами каждый
- Силомоментная обратная связь (датчики давления FSR-402)
- Бесконтактный ИК-датчик обнаружения объекта
- Светодиодная индикация состояния
- Два режима управления: Bluetooth (со смартфона) и USB (с ПК)

## 2. Механическая конструкция и материалы

| Элемент | Описание |
|---|---|
| **Корпус** | Ладонь и фаланги напечатаны на 3D-принтере из пластика PETG (повышенная прочность и термостойкость) |
| **Сборка** | Все детали соединяются винтами M3 с потайной головкой и самоконтрящимися гайками — конструкция разборная и ремонтопригодная |
| **Тяга** | Стальная рыболовная проволока (струна) диаметром 0,4 мм, протянута через каналы внутри фаланг |
| **Возврат** | Пружины растяжения из нержавеющей стали на тыльной стороне каждого пальца |
| **Шарниры** | Металлические оси диаметром 2 мм с латунными втулками — без люфта |
| **Защита** | Силиконовые накладки на кончиках пальцев для увеличения трения |

## 3. Электронная архитектура и обратная связь

### Контроллер

**Arduino Mega 2560** — выбран из-за большого количества ШИМ-каналов (15) и аппаратных таймеров для одновременного управления тремя сервоприводами.

### Датчик усилия (силомоментная обратная связь)

**FSR-402** — тензорезистивные датчики давления, закреплённые на внутренней стороне кончиков пальцев.

**Принцип работы:**

1. При захвате предмета сопротивление датчика падает пропорционально силе
1. Контроллер считывает аналоговый сигнал с делителя напряжения
1. При достижении порогового значения сервопривод останавливается
1. Защита хрупких предметов от повреждения

### Датчик касания

**TCRT5000** — инфракрасный датчик препятствия в центре ладони. Срабатывает при приближении объекта на расстояние < 2 см, запуская алгоритм автоматического захвата.

### Индикация (RGB-светодиод)

| Цвет | Состояние |
|---|---|
| 🔵 Синий | Ожидание команды |
| 🟢 Зелёный | Захват выполнен успешно |
| 🔴 Красный | Превышение усилия / ошибка |

### Связь

- **HC-05 Bluetooth** — беспроводное управление со смартфона
- **USB-UART** (встроен в Arduino Mega) — отладка и управление с ПК

## 4. Система управления

### Архитектура управления

```
СМАРТФОН/ПК          ARDUINO MEGA           ИСПОЛНИТЕЛЬНЫЕ
(команды)     →      (обработка)     →      УСТРОЙСТВА
    │                    │                      │
    │  Bluetooth/USB      │  ┌─────────────────┤
    ├───────────────────→│  │ • Серво MG995 ×3 │
    │                    │  │ • RGB-светодиод  │
    │                    │  └─────────────────┤
    │                    │                     │
    │                    │  ДАТЧИКИ            │
    │                    │  ┌─────────────────┤
    │                    │←─│ • FSR-402 ×3     │
    │                    │  │ • TCRT5000       │
    │                    │  └─────────────────┤
    │  ◄─────────────────│                     │
    │   статус/данные    │                     │
```

### Алгоритм захвата

1. **Ожидание:** Синий светодиод, сервоприводы в нейтральном положении (пальцы раскрыты)
1. **Обнаружение:** ИК-датчик фиксирует объект → переход к шагу 3
1. **Захват:** Сервоприводы начинают сгибать пальцы с постоянной скоростью
1. **Контроль усилия:** Опрос датчиков FSR каждые 10 мс
1. **Остановка:** Как только любой из датчиков достигает порога → сервоприводы останавливаются
1. **Фиксация:** Зелёный светодиод, захват удерживается
1. **Отпускание:** По команде пользователя пальцы возвращаются в исходное положение

### Режимы управления

| Режим | Интерфейс | Описание |
|---|---|---|
| **Ручной** | Bluetooth (приложение) | Пользователь управляет каждым пальцем отдельно (согнуть/разогнуть) |
| **Автоматический** | Bluetooth (приложение) | При обнаружении объекта — автоматический захват |
| **Отладка** | USB (Serial Monitor) | Вывод данных с датчиков, ручная установка параметров ШИМ |

### Протокол команд (Bluetooth)

| Команда | Действие |
|---|---|
| `G` | Захват (автоматический режим) |
| `R` | Отпустить (разжать пальцы) |
| `S` | Статус (запросить состояние датчиков) |
| `1-3` `0-180` | Установить угол пальца (например, `2 90` — палец 2 на 90°) |

## 6. Веб-интерфейс переключателя

В рамках дополнительного задания разработан веб-интерфейс переключателя (toggle switch) с индикацией состояния.

**🔗 Ссылка на сайт:** [https://N1ghtmareDe.github.io/my-switch-site/](https://N1ghtmareDe.github.io/my-switch-site/)

**Функционал:**

- Переключатель «включено/выключено» с анимацией
- Светодиодный индикатор состояния (красный/зелёный)
- Счётчик количества переключений
- Таймер учёта времени во включённом состоянии
- Поддержка клавиатуры (Tab + Enter/Пробел)
- Адаптивный дизайн (работает на телефоне и ПК)

**Технологии:** HTML5, CSS3, JavaScript (архитектура в стиле ООП C++: класс `PowerController` с инкапсуляцией, конструктором и деструктором).
